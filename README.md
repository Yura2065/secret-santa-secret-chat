# Secret Santa Platform

Вебзастосунок для організації святкового обміну подарунками. Платформа вирішує проблеми класичного жеребкування, пропонуючи автоматизований розподіл учасників, анонімне спілкування та управління статусами подарунків. Проєкт створено для забезпечення зручного інструменту швидкого формування груп без ризику передчасного розкриття результатів.

## Основні можливості

1. Розумний алгоритм жеребкування (Smart Matching Engine)
Математичний алгоритм автоматичного розподілу пар з урахуванням взаємних обмежень учасників. Дозволяє організаторам налаштовувати винятки, щоб певні люди не могли дарувати подарунки одне одному.

2. Анонімний чат
Інтегрована система безпечного обміну повідомленнями. Дарувальник може анонімно спілкуватися зі своїм підопічним для уточнення деталей щодо подарунка, зберігаючи при цьому інтригу до самого свята.

3. Відстеження статусу подарунка
Кожен учасник має змогу змінювати статус свого подарунка (наприклад, "Готується" або "Отримано"), що дозволяє організаторам контролювати процес та впевнитися, що ніхто не залишиться без сюрпризу.

4. Панель адміністратора
Зручний інтерфейс для модерації ігрових кімнат. Організатор може легко додавати або видаляти учасників, керувати правилами та відстежувати загальну активність групи.

5. Захист персональної інформації
Реалізовано надійні механізми захисту даних на рівні окремих рядків таблиць бази даних. Це гарантує повну конфіденційність жеребкування.

## Технологічний стек

Проєкт побудовано з використанням сучасних вебтехнологій для забезпечення максимальної швидкодії та безпеки.

* Клієнтська частина: React, TypeScript, Vite
* Візуальне оформлення: Tailwind CSS, компоненти shadcn/ui
* База даних та серверну частину реалізовано на Supabase (PostgreSQL)

## Локальне розгортання та запуск

Для запуску проєкту на вашому комп'ютері переконайтеся у наявності встановленого середовища Node.js.

Крок 1. Зробіть клон репозиторію
git clone <URL_репозиторію>

Крок 2. Відкрийте папку проєкту
cd secret_santa

Крок 3. Встановіть необхідні залежності
npm install

Крок 4. Запустіть сервер для розробки
npm run dev

## Архітектура та структура бази даних

Застосунок використовує реляційну базу даних PostgreSQL, розгорнуту в хмарному середовищі Supabase. Архітектура передбачає чітке розділення прав доступу, де кожен користувач має доступ виключно до необхідної йому інформації. Алгоритм перемішування списків працює на стороні сервера, що повністю виключає можливість втручання або маніпуляцій з результатами жеребкування.
